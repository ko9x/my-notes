Git And GitHub Notes



MISC GIT NOTES

HEAD

This refers to the most recent commit of the current branch.

IGNORE A FILE SO IT WONT GET TRACKED OR COMMITTED

To ignore a file add it to the .gitignore file

CHANGING A FILE NAME WITH GIT

git mv “filename” “newfilename”

This will change the name of the file and keep it tracked by git. But you need to make sure to commit this change because using the mv command does not commit that change to git until you actually do so using git commit -m “what ever comment you want”

REMOVING A FILE WITH GIT

git rm “filename”

This command will remove the file from the directory and git will track that the file has been removed. But you must commit this change before git will actually update that it has been removed. So make sure to run git commit -m “comment about deleting the file”



GIT COMMANDS

git cat “filename”

This will show the contents of the file named filename in the terminal.

git init .

This will add git to the directory you are currently in

git status

This will show you what has been modified since the last time you made a commit

git ls-files

This command will show you all the files that git is tracking

git add “filename”

This will allow you to add a particular folder or file to a commit

git add .

This will add everything that has been modified since the last commit to the current commit.

git reset HEAD “filename”

This command will remove the changes that were added to the staging area. It won’t actually modify the file that was added but it will just forget that you had added it to the staging area. This is if you added it accidentally or just wanted to make some other change before adding to the staging area and then commiting it.

git checkout -- “filename”

This command WILL modify the file that was added and revert it back its state at the last commit.



COMMIT COMMANDS

git commit

This will open the editor and let you type your commit message in a large window that shows all the files included in the commit

git commit -m “add a comment explaining what is new"

This will add an comment to the commit without needing to open the window and add the commit there.

git commit -a

This will add all the tracked files and any changes made to them to the commit and then open a window so you can add your comment in the editor

git commit -am “add commit message here”

This will let you add all the modified files to the commit and add a comment without needing to open the editor window



LOG COMMANDS

git log

This will show you all the commits that you have made.

exit git log

To exit git log press q

git log --oneline

This will show the log on single lines

git reflog

This will all the different actions that have been taken in the repository. this can be a pretty long list so only use it if you need it.

git show

This will show the most recent commit message, the files that were added to the commit and what modifications were made to those files.

git show “full sha id”

This will show you all the details about a specific commit

exit show

To exit git show press q

git hist

This is a git shortcut that I made using git alias this command will show you a condensed history of all the commits to the project you are currently in.

git hist -- “filename”

This command will give a condensed history of all the commits that included the selected file.

git hist creation code

This is how git hist was created git config --global alias.hist "log --oneline --graph --decorate --all"



DIFF COMMANDS SECTION

git diff

This command will show you all the differences between the most recent commit and all the files in the directory you are in

git diff (a git reference number) HEAD

This command will show you all the differences between the reference numbers commit and the most recent commit

git difftool

This command will open the p4merge window and make it easier to see what is different and I think you can use it to modify the files but I have not done that yet. you can use difftool or diff in place of each other, diff will show the data in the terminal and p4merge will show it in an easier to read window.



BRANCH SECTION

git branch

This command will show you what branch you are on

git branch -a

This command will show you all the branches that the local directory is aware of.

DELETE A BRANCH

git branch -d “branch name”

This command will delete the branch

IMPORTANT NOTE

After deleting a branch push the changes to the repo with

git push

And then run

git fetch -p

This will remove any reference to deleted branches

SIDE NOTE

The above command is usually used to clear out remaining records of branches that have been removed from the remote repo and then that change was fetched.

DELETE A REMOTE BRANCH FROM THE TERMINAL

git push origin :branch name

Make sure to add the colon and then type the name of the remote branch you want to delete.

IMPORTANT SIDE NOTE

This is pretty much only going to be done after you delete a branch in the local repo and then want the remote repo to be updated with that change (the deleted branch)

SWITCH TO A DIFFERENT BRANCH

git checkout “branch name”

This command will switch the HEAD to whatever branch you specified

CREATING A NEW BRANCH AND SWITCHING TO IT

git checkout -b “branch name”

This will create a new branch and also move the HEAD to this new branch. essentially moving you into a new branch and out of the master branch

SWITCHING BACK TO THE MASTER BRANCH

git checkout master

This will switch the HEAD back to the master branch



MERGING COMMANDS

git merge “branch name”

If it is a simple merge it will use the fast forward, for more complex merges it may be a little more difficult to merge them. But I’ll get to that later.

git mergetool

This will open the p4merge window and show you the merging files. Use this command if you encounter a merging error or failure.



TAG COMMANDS

git tag

This command will list all of the tags for the current directory

git tag “tag name”

Just makes a light-weight tag that doesn’t have any associated data but just places that tag on the current commit.

git tag “tag name” “commit ref”

This will make a light-weight tag on the commit that is specified with the tag ref id

git tag -a “tag name” -m “a message or note about the tag”

This is an annotated tag. It will provide more information about why the commit was tagged.

SIDE NOTE

Use git show “tag name” to see the information.

git tag -a “tag name” -m “a message or note about the tag” “ref”

This will make an annotated tag at the commit that is referenced at the end of the command

git tag -d “tag name”

This command will delete the tag named “tag name”

SIDE NOTE

You have deleted the tag locally you can delete it from the remote repo by using this command

git push origin :tag name

ADDITIONAL SIDE NOTE

After updating/moving the tag on the local directory you will most likely want to update the remote repo with that change

git push —force origin “tag name”

This will force the remote repo to update the tag. (you need to use the force flag because git sees a tag with that name already exists and doesn’t want to do anything to change it)

git tag —list

This will list all the tags

UPDATE AN EXISTING TAG

git tag -f “tag name” “commit ref”

This will move the existing tag to the new commit ref specified.

SIDE NOTE

Just like most git commands, if you don’t add a commit ref to the end of the command it will assume you are moving the tag to the HEAD position. (the most recent commit)

PUSHING TAGS TO GITHUB

git push “tag name”

This command will push the specified tag to the remote repository

git push —tags

This command will push all the tags to the remote repository



STASH COMMANDS

git stash

This will allow you to stash some changes that have not been added yet, so you can add and commit them at a later point.

git stash list

This will show you the stashes and where they were made

git stash pop

This will basically unstash your stashed changes so you can then add and commit them.



TIME-TRAVEL/RESET COMMANDS

git reset (some previous commit id) --soft

This will move the HEAD to the commit point specified and leave all the uncommitted files as they were, for example if I had a file that was in the staging area and another that had not been added to the staging area at the time I did the soft reset, they would be in the same places. keep in mind any files you commit in this new HEAD location will be saved in this new HEAD location. seems obvious but I wanted to make a note of it.

git reset (some previous commit id)

This will run the default reset which is —mixed The HEAD will be moved to the commit that was specified and any staged files will now be moved out of the staging area and need to be added again in order to be committed. keep in mind any files you commit in this new HEAD location will be saved in this new HEAD location. seems obvious but I wanted to make a note of it.

git reset (some previous commit id) --hard

The HEAD will be moved to the commit that was specified and this will remove any staged or unstaged files at the point the command was run. only run this type of reset if you are sure you don’t care about the data that has not been committed at the point the command was run. keep in mind any files you commit in this new HEAD location will be saved in this new HEAD location. seems obvious but I wanted to make a note of it.

git reflog

This command is used to see all the actions that have been taken and make sure you put the HEAD in the right place.



REMOTE GIT COMMANDS SECTION

git remote -v

This will show any remote connections to the project you are currently in.

git push -u origin master

or

git push -u origin master —tags

(Only add —tags if you have some tags in the project already) This will update the repository on github so make sure you are ready to save those changes to the remote repository before you enter this command

SIDE NOTE

You only need to add the -u the first time you push to github

git push

Once you have your repository set up you can just use the git push command. This command will just push the updated files on the branch that you are on. So if you want to update multiple branches you will need to navigate to each branch and push them individually.

git fetch

This command will retrieve all the changes on the remote repository and integrate them into the local repository. So its like the reverse of a git push.

SIDE NOTE

You also use git fetch to update the local directory with any branches that have been added to the remote repo. BUT to add a new branch that was created on the remote you’ll need to check it out. Run this command after fetching the branch

git checkout “new branch name”

Git will check your local directory for a branch with that name, if it doesn’t find one it will then check the remote repo. You will be automatically switched to the new branch,

any changes that you make can be updated on the remote new branch by running

git push

Just make sure you are still in the new branch or else the changes will get push to whatever remote branch corresponds to the branch you are in when you run the push command.

git pull

This command will basically do a fetch and then merge the data between the remote repo (github) and the local directory in which we are working.

IMPORTANT SIDE NOTE

You will need to do a git push after the pull is complete to update the remote repo and save the changes made by the pull command to the remote repo.

git pull --all

This command will update all the branches that are being tracked with the data on the remote repo. git pull will just get the data corresponding to the branch you are currently in.

Changing the name of the remote repository

If you decide to change the name of the remote repo you will need to copy the clone link again, navigate to the directory in terminal and run this command git remote set-url origin “paste the url here without quotes”

SIDE NOTE

Use git remote -v to double check you have linked the correct remote repo to the project directory.

ADDING A BRANCH ON THE LOCAL DIRECTORY THEN PUSHING TO THE REMOTE REPO

git checkout -b “new branch name” (don’t add quotes)

This will create a new branch with whatever name give it. once you have created the branch and you want to push it to the remote repo you will need to run

git push -u origin “new branch name”

The -u sets up the tracking for the new branch on the remote repo
